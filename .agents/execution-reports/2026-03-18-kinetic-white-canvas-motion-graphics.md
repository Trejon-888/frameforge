## Problems Solved

**White background pixelation bug:** Creating a separate `white-bg.mp4` via FFmpeg lavfi outputs 17 kb/s video. FrameForge quality-matches output bitrate to input → 244 kb/s → severe pixelation. Fix: always use the source footage as render-overlays input (for quality reference). Add `background:#fff; position:fixed; width:1080px; height:1920px` overlay as overlay[0]. Puppeteer captures it as fully opaque white pixels (omitBackground:true), FFmpeg composites to completely cover source video.

**Canvas 2D transparency:** `ctx.clearRect(0,0,W,H)` (not `ctx.fillRect`) makes cleared areas transparent in Puppeteer PNG capture — correct for compositing layers.

**SVG GSAP attr animation:** GSAP animates SVG attributes via `{attr:{r:value}}` syntax — enables circle ring radius expansion.
