# Plan: Social Media Launch Content

**Status:** Ready
**Priority:** P1 — Awareness after npm publish
**Depends on:** npm publish + README

---

## Objective

Create 5-8 showpiece videos that demonstrate FrameForge's unique capabilities. Each video serves as both a demo AND social media content. Post to Twitter/X, LinkedIn, YouTube, and relevant dev communities.

---

## Content Strategy

### Core Message
"If a browser can render it, FrameForge can record it. No React lock-in. No framework required."

### Content Pieces

| # | Title | What It Shows | Platform | Duration |
|---|-------|--------------|----------|----------|
| 1 | **"Any Framework"** | Same animation in GSAP, Three.js, CSS, vanilla JS — all rendered by FrameForge | Twitter/X | 30s |
| 2 | **"3 Lines to Video"** | Screen recording: write SDK code → render → play MP4 | Twitter/X | 15s |
| 3 | **"Studio Tour"** | Screen recording of Studio: scrub timeline, edit code, hot-reload, render | YouTube | 60s |
| 4 | **"Remotion vs FrameForge"** | Side-by-side: React-only vs any-framework comparison | Twitter/X | 30s |
| 5 | **"AI Makes a Video"** | Claude Code + Agent Skill generates a video from a text prompt | Twitter/X | 45s |
| 6 | **"Python to Video"** | Python SDK: define scene → render → MP4 (no JS required) | Twitter/X | 20s |
| 7 | **"Multi-Scene in 10s"** | Composition with transitions — 3 scenes stitched with wipes | Twitter/X | 15s |
| 8 | **"Data Viz Animation"** | Animated bar chart from raw data → polished MP4 | LinkedIn | 30s |

### Production Notes
- All videos rendered BY FrameForge (meta: the tool demos itself)
- Dark aesthetic, brand colors (#e94560, #6366f1)
- Include `npm install @frameforge/core` at the end of each
- Use brand video scene3 (FF monogram + CTA) as outro on all
